1. 簡介
    2.

    當我們在儲存一筆同型資料時，通常都是使用較簡單的「陣列(Array)」來進行儲存，但是我們在宣告陣列時，都需要明確指定其大小，也因為如此，如果資料數 ＜ 陣列大小，就會造成記憶體上的浪費；而資料數 ＞ 陣列大小時，則會使得記憶體的空間不夠。


    為了能讓資料妥善配置在記憶體中，「鏈結串列(Linked List)」便發明了！Linked List是串列的一種，由許多節點所組成，也是一種常見的資料結構，它最大特色就是可以動態配置記憶體，能在不使用連續的記憶體空間，存著一筆連續的資料，不過在實作上就比陣列還要麻煩了。




  2. 特性
    3.

    • 動態配置記憶體

    • 修改指標容易，彈性插入或移除節點在串列首、尾、中間

    • 利用Linked List可更進一步實作出其他資料結構

      Ex. 堆疊(Stack)、佇列(Queue)



  3. 比較
    4.

    Array 儲存方式 Linked List 儲存方式
    實作 較簡單，只要宣告即可，但需要事先宣告陣列大小，造成記憶上的浪費。 每個節點除記錄資料外，還需要用額外的指標指向下一個節點，不需事先指定串列大小。
    各筆資料型態 需全部相同 可不用全部相同
    資料的加入與刪除 需做大量的資料搬移 可動態增加、刪除空間，修改指標即可。
    記憶空間 占用連續的記憶體空間 可以非連續，但需額外記憶體空間紀錄節點指標。
    資料的存取 可以循序或隨機存取 隨機存取功能弱，可以循序存取，但又需要先讀取指標，所以存取速度慢。
    資料的搜尋速度 快，陣列可應用至二分搜尋(Binary Search)。 慢，需走訪(Traversal)各個節點。
    可靠度 高 低，因為當指標斷裂，資料即遺失。
